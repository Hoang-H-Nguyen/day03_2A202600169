## III. Personal Insights: Chatbot vs ReAct (10 Points)

*Reflect on the reasoning capability difference.*

1.  **Reasoning**: How did the `Thought` block help the agent compared to a direct Chatbot answer?
    - Thought: giúp mô hình suy nghĩ trước khi hành động, agent có thể chia nhỏ bài toán, giảm tình trạng trả lời bừa, dễ  debug hơn vì ta có thể  thấy được từng bước suy nghĩ của model.
2.  **Reliability**: In which cases did the Agent actually perform *worse* than the Chatbot?
    - Khi model không tuân thủ format, model có thể dẫn đến bị lỗi hoặc lặp.
    - Câu hỏi đơn giản chatbot trả lời ngay sẽ nhanh và ổn định hơn, ReAct thì phải trải qua nhiều bước không cần thiết.
3.  **Observation**: How did the environment feedback (observations) influence the next steps?
    - Giúp agent cập nhật trạng thái, quyết định bước tiếp theo. Nếu tool trả về dữ liệu, agent có thể chuyển sang final answer, nếu không, tool có thể trả về lỗi không tìm thấy. agent có thể thử lại với cách khác. Tuy nhiên, nếu observation không rõ ràng, model có thể hiểu sai dữ liệu, hoặc không có cấu trúc. Vì vậy, chất lượng của Observation rất quan trọng, cần có cấu trúc, format rõ ràng.

## IV. Future Improvements (5 Points)

*How would you scale this for a production-level AI agent system?*

- **Scalability**: [e.g., Use an asynchronous queue for tool calls]
    Tách agent thành các service độc lập theo kiến trúc microservice.
    Thêm caching (Redis) cho các truy vấn phổ biến.
- **Safety**: [e.g., Implement a 'Supervisor' LLM to audit the agent's actions]
    Triển khai một supervised LLM hoặc rule-based layer để kiểm tra: action có hợp lệ không, có gọi tool sai mục đích không.
- **Performance**: [e.g., Vector DB for tool retrieval in a many-tool system]
    Sử dụng vector database để tìm tool phù hợp trong nhiều tool. Tối ưu prompt ngắn gọn để tiết kiệm token.
    Dùng model routing: model nhỏ cho task đơn giản, model mạnh cho task phức tạp.
